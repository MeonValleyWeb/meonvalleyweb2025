# MeonValleyWeb Refactor Plan

> Make the site hosting-accurate, slim, accessible, and SEO-friendly.

**Goal:** Update the Meon Valley Web site so hosting details are correct, the design is clean and consistent, the codebase is slim and refactorable, and the site passes a full accessibility audit while retaining (and improving) SEO performance.

**Tech Stack:** Astro 6.x, Tailwind CSS 4.x, static export to Netlify, Netlify Forms, Fathom + Umami + Google Ads.

**Current working branch:** `feature/conversion-seo-improvements` (clean working tree at scan time).

---

## 1. Discovery Summary

### 1.1 Current state
- **Framework:** Astro 6.1.1 with Tailwind 4.2.2, `@astrojs/sitemap`, `@tailwindcss/typography`, `marked`.
- **Inputs:** ~10k Astro lines across 37 `.astro` files + ~1.6k lines of Markdown content.
- **Public assets:** ~92 MB in `dist/`, 87 MB in `public/`, with many Unsplash / marketing images at 2–5 MB.
- **Deploy target:** Netlify (deployed from `dist/`).

### 1.2 Hosting / service accuracy issues
- Page says "24/7 Support" but no hours / escalation path is defined.
- Pricing is duplicated across `services.astro`, `hosting.astro`, and `src/pages/archived/pricing.astro`.
- Hosting plan name "Wayfarer" / "Shipwright" / "Monarch" are defined in multiple places; real hosting stack reference (SpinupWP, Vercel, Netlify) is inconsistent.
- No single source of truth for pricing or plan features.
- Contact form uses Netlify Forms with a honeypot; thank-you pages exist but form feedback for screen readers is absent.

### 1.3 Design bloat / maintainability issues
- ~3,243 `class="..."` attributes, ~12,500 class tokens, ~397 unique classes.
- Pages are mostly copy-paste HTML with duplicated card / feature / pricing grids.
- 10 inline `<script>` tags scattered across pages and components.
- 296 inline SVG elements, many duplicated for checkmarks and icons.
- Agressive use of `shadow-lg`, `rounded-xl`, gradients, and decorative blobs.
- Hero section uses JavaScript to synchronise image height with content height.
- Exit-intent popup (`LeadMagnetPopup`) uses `mouseout` / `clientY` tracking that can fail on keyboard-only users and mobile users.

- Duplicated filenames: Unsplash JPEGs appear in both `public/` and `public/portfolio/marketing/`.
- Build time and output size will keep growing as location/marketing pages expand.

## 2. Guiding Principles

1. **Single source of truth:** pricing, hosting plans, service features, contact details, and business metadata come from one content/config file each.
2. **Component-driven:** convert duplicated HTML islands into reusable Astro components.
3. **Accessible by default:** keyboard navigation, screen-reader feedback, colour contrast, and focus management built into components.
4. **Progressive enhancement:** interactive bits (menu, popup, form feedback) work without JS and are enhanced when JS loads.
5. **Asset diet:** remove unused/duplicate assets, compress remaining images, use Astro `Image` with responsive `sizes`.
6. **No big-bang rewrite:** keep existing URLs and Netlify Forms working; refactor page by page.

### Phase 4 — Accessibility Audit & Remediation
**Goal:** achieve WCAG 2.1 AA across the whole site.

- [ ] **P4.1 Automated scan baseline.**
  - Run `pa11y-ci` against all built pages and capture the report.
  - Run Lighthouse accessibility on homepage, contact, services, hosting, one location, one blog post.

- [ ] **P4.2 Heading structure fix.**
  - Normalize heading outline on every page starting with `h1` and increasing by no more than one level.
  - Fix `upgrading-astro-5-to-6.md` dual `h1` issue.
  - Use `SectionHeading` component everywhere to enforce levels.

- [ ] **P4.3 Colour contrast sweep.**
  - Replace `text-gray-400` body text with `text-gray-600` minimum.
  - Ensure primary pink `#BC1F63` on white is ≥ 4.5:1 (it is; keep).
  - Ensure any text on coloured backgrounds passes.

- [ ] **P4.4 Keyboard & focus fixes.**
  - Verify every interactive element has visible focus.
  - Add focus trap and `Esc` close to `LeadMagnetPopup`.
  - Ensure popup is not displayed to `prefers-reduced-motion` and keyboard users as a blocking overlay on load.
  - Convert popup trigger to an optional "Get the checklist" button rather than exit intent.

- [ ] **P4.5 Form accessibility.**
  - Add `<div aria-live="polite" role="status" id="form-status">` to contact form (and lead magnet form) for success/error announcements.
  - Link inputs to errors with `aria-describedby` when validation fails.
  - Ensure Netlify success/error pages have clear headings and return links.

- [ ] **P4.6 Screen-reader landmarks / announcements.**
  - Add `aria-label` to `<nav>` and repeated `<section>` blocks.
  - Add descriptive link text; avoid "click here", "Read more".
  - Mark current page in nav/pagination/breadcrumbs with `aria-current="page"`.

- [ ] **P4.7 Manual audit checks.**
  - Test with keyboard-only navigation through every page template.
  - Test with macOS VoiceOver on homepage and contact form.
  - Document findings in `a11y-audit-report.md`.

**Acceptance:** Lighthouse accessibility score 100 on all tested templates; `pa11y-ci` passes with 0 errors; manual keyboard/VoiceOver checks complete.

## 4. Risks & Trade-offs

| Risk | Mitigation |
|---|---|
| Removing Unsplash hero images makes pages feel generic. | Replace with one high-quality, brand-relevant optimised image per main page; keep rest minimal. |
| Tailwind refactor accidentally breaks visual parity. | Migrate one section at a time, build+visual-compare after each phase. |
| Pricing changes may conflict with real quotes. | Make `site.ts` the canonical source; any custom quote exceptions live in copy, not in data. |
| Large redirect surface on `pricing.astro`. | Add a 301 redirect in `_redirects` before deleting the page. |
| Over-optimising for SEO creates keyword stuffing. | Keep copy human-first; add schema and metadata, not repetitive text. |
| Accessibility changes affect LeadMagnetPopup conversion. | Make popup opt-in button-driven by default; A/B test exit-intent later. |
